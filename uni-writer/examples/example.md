# **Ejercicio 2: Benchmarking: investigación de la competencia**

|  | LibriVox | SerialReader | GoodReads |
| :---- | :---- | :---- | :---- |
| **1\. Descubrimiento y Recomendaciones** | **Positivo:** Tiene un catálogo muy grande de audiolibros gratis, lo que permite encontrar clásicos por autor o género fácilmente. **Negativo:** No tiene un sistema que te recomiende libros según lo que has escuchado antes o tus gustos. | **Positivo:** El formato de leer por entregas diarias ya es una forma de recomendar clásicos de manera amena. **Negativo:** Los filtros para buscar libros son muy simples; no puedes filtrar por temas, duración o dificultad. | **Positivo:** Lo mejor son las recomendaciones de la gente, que te permiten descubrir libros por las listas y críticas de otros. **Negativo:** Las recomendaciones automáticas del sistema no explican por qué te sugieren ese libro y no son muy personales. |
| **2\. Accesibilidad y Adaptabilidad** | **Positivo:** Al ser audiolibros, es fácil de usar para personas con problemas de vista o mientras se hacen otras cosas. **Negativo:** El reproductor es muy básico; le faltan opciones como cambiar la velocidad, poner un temporizador o saltar hacia adelante/atrás. | **Positivo:** Los fragmentos cortos se leen muy bien en el transporte público y permiten usar el móvil con una sola mano. **Negativo:** Casi no puedes personalizar la lectura; no deja cambiar el tipo de letra, el espacio entre líneas o el contraste. | **Positivo:** Permite crear estanterías propias para organizar los libros como cada uno quiera. **Negativo:** No es una aplicación para leer, por lo que no tiene opciones para adaptar el texto o el audio por sí misma. |
| **3\. Continuidad y Retención** | **Positivo:** La app guarda exactamente por dónde te quedaste para que puedas seguir escuchando sin problemas. **Negativo:** No tiene retos ni avisos para animarte a seguir, por lo que depende solo de las ganas del usuario. | **Positivo:** Las notificaciones diarias ayudan a crear el hábito de leer un poco cada día. **Negativo:** No tiene un “modo vacaciones”, así que si un día no lees se te acumulan las entregas y parece un deber. | **Positivo:** El reto de lectura anual motiva mucho a seguir leyendo para cumplir el objetivo que te has puesto. **Negativo:** Como es muy social, si dejas de usarla un tiempo parece que hay un vacío en tu perfil y no hay forma de avisar de una pausa. |
| **4\. Calidad y Control** | **Positivo:** Al ser libros grabados por voluntarios, avisan de que no son profesionales. No tiene anuncios. **Negativo:** La calidad de las grabaciones varía mucho y a veces los datos de los capítulos no están claros. | **Positivo:** El contenido está bien seleccionado y preparado para leerlo por entregas, con una presentación cuidada. **Negativo:** No siempre dicen qué edición o traducción exacta están usando para el libro. | **Positivo:** Los datos de los libros (edición, páginas, etc.) suelen estar muy completos gracias a la comunidad. **Negativo:** Al ser gratis, a veces los anuncios o los correos de publicidad pueden ser molestos si no se configuran bien. |
| **5\. Sistema de Valoración** | **Positivo:** No tiene sistema de notas, lo que permite disfrutar del libro sin la presión de tener que dar una opinión. **Negativo:** Al no haber críticas, no puedes saber si la narración es buena o mala antes de empezar a escuchar. | **Positivo:** Al terminar un libro, te deja dar una nota sencilla sin complicarte con muchas opciones. **Negativo:** Es difícil evitar los “spoilers” en las pocas charlas que hay y no se pueden filtrar las críticas. | **Positivo:** El sistema para ocultar “spoilers” en las reseñas funciona muy bien para no estropear el libro a otros. **Negativo:** Las notas de 1 a 5 estrellas se quedan cortas para valorar detalles como el ritmo o la dificultad del libro. |

##  **Ejercicio 3: Aspectos básicos de Interacción Persona Ordenador** 

### **1- Metáfora**

Usar objetos o ideas del mundo real en el diseño digital para que el usuario entienda rápido cómo funciona el programa.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | Usa la idea de una biblioteca de audio para organizar los libros en estantes y catálogos, como si estuviéramos en una biblioteca de verdad. |
| **SerialReader** | Se basa en las novelas que se publicaban por capítulos en los periódicos antiguos, mandando un trozo del libro cada día. |
| **GoodReads** | Funciona como un club de lectura digital donde puedes compartir lo que lees y hablar de libros con otras personas. |

###  **2- Affordance**

**Definición:**  
Pistas visuales que te dicen para qué sirve un botón o elemento antes de tocarlo. El diseño te indica “cómo se usa”.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | El icono del triángulo es una señal clara en todo el mundo de que si lo pulsas, el audio empezará a sonar. |
| **SerialReader** | La barra que se va llenando indica cuánto has leído y te da a entender que el capítulo tiene un final. |
| **GoodReads** | El botón de “Quiero leer” destaca sobre lo demás para que sepas que ahí es donde guardas los libros en tu lista. |

### 

### **3- Visibilidad**

Que el estado de la aplicación y las opciones disponibles se vean fácilmente. Ayuda a que el usuario no tenga que adivinar dónde están las cosas.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | En la pantalla principal aparecen directamente los libros que has escuchado hace poco para que los tengas a mano. |
| **SerialReader** | Al entrar ves enseguida qué libros estás leyendo y cuánto te falta en cada uno de ellos. |
| **GoodReads** | En el muro principal ves lo que están haciendo tus amigos, dejando claro que es una plataforma para interactuar. |

### 

### **4- Retroalimentación**

La respuesta que te da el sistema para confirmar que ha hecho lo que le has pedido. Te avisa de si algo ha salido bien o si hay un error.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | Cuando terminas un libro, la aplicación le pone una marca para confirmar que ya lo has completado. |
| **SerialReader** | Al acabar la lectura del día, te sale un mensaje de felicitación y te dice cuándo estará lista la siguiente parte. |
| **GoodReads** | Al mover un libro a una estantería, el botón cambia y el libro aparece al momento en tu lista, confirmando el cambio. |

### 

### **5- Restricción**

Límites que pone el diseño para evitar que el usuario se equivoque o para guiarlo por el camino correcto.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | No te deja intentar descargar o escuchar un libro si detecta que no tienes conexión a internet. |
| **SerialReader** | Bloquea el acceso al siguiente capítulo hasta que pasen 24 horas, para mantener el ritmo de lectura diaria. |
| **GoodReads** | Solo te deja dar una nota entre 1 y 5 estrellas, impidiendo que pongas puntuaciones que no encajen. |

### 

### **6- Modelo Mental**

Lo que el usuario cree que va a pasar basándose en su experiencia con otras aplicaciones. El diseño debe encajar con lo que el usuario ya sabe.

| Aplicación | Implementación |
| :---- | :---- |
| **LibriVox** | Como se parece a cualquier otra aplicación de música, ya sabemos usar los botones de pausa y avanzar sin que nadie nos explique. |
| **SerialReader** | Funciona de forma parecida a un calendario o una agenda, por lo que entendemos rápido que hay que entrar cada día. |
| **GoodReads** | Al usar iconos y funciones parecidas a Facebook o Instagram, cualquier persona que use redes sociales sabe manejarse enseguida. |
